# 사업 한 줄 정의

**연애가 목적인 걸 배경에 깔되, 활동(보드게임·운동·봉사 등) 중심으로 설계된 소셜 모임 서비스**

"소개팅은 피곤하고, 데이팅앱은 가볍다"는 공백을 파고드는

# 1. 왜 지금인가

## 시장 구조의 변화

```
오프라인 만남 → 온라인 만남 비율 역전
개인화 심화 → 주변의 소개 문화 축소
결과: 만남 채널이 절대적으로 부족하다
```

## 기존 대안의 문제

| 채널 | 문제 |
|---|---|
| 데이팅앱 (틴더, 범블) | 외모 중심, 대화 지속 어려움, 가볍다는 인식 |
| 소개팅 | 목적 노출 → 매 만남 리셋, 대화 이끌어야 하는 피로 |
| 트레바리/독서모임 | 콘텐츠는 있으나 연애 목적 금기 문화 |
| 클럽/파티 | 너무 가볍고 장기 연결 불가 |

## 공백

**연애가 목적이지만, 활동이 있어서 부담이 없는** 포맷이 없다.

# 2. 타깃 고객

## 프라이머리 타깃

다음 조건을 동시에 충족하는 사람

- 30대 초중반 미혼 직장인 (서울/수도권 기준)
- 소개팅 경험 있으나 피로감 높음
- "좋은 사람 만나고 싶지만 방법이 없다"는 상태
- 취미 활동에 관심 있음 (운동, 보드게임, 문화 생활)

## 세컨더리 타깃

- 20대 중반 (대학 졸업 후 사회 초년생)
- 장기: 돌싱, 황혼 시니어 세그먼트 (Phase 3+)

## 왜 이 타깃인가

| 조건 미달 시 | 문제 |
|---|---|
| 20대 초반 | 연애 진지도 낮음 → 문화 유지 어려움 |
| 결혼 목적 강한 40대+ | 포맷 미스매치 |
| 연애 목적 없음 | 커뮤니티 문화 훼손 |

# 3. 고객의 문제

## 핵심 문제

**소개팅은 목적이 명확해서 오히려 부자연스럽다.**

에이전시는 만남을 주선하고, 사람이 원하는 것은 자연스러운 연결이다.
이 둘이 연결되어 있지 않다.

## 레이어별 문제

| 문제 | 참가자의 실제 언어 |
|---|---|
| 리셋 피로 | 매번 처음부터 대화해야 해, 지쳐 |
| 대화 부담 | 한 명이 계속 이끌어야 하는데 그게 나야 |
| 평가 압박 | 짧은 시간에 다음 만남 여부 결정해야 해 |
| 맥락 부재 | 그 사람에 대해 아는 게 없어, 어색해 |

## 우리 솔루션이 작동하는 이유

활동이 있으면

- 다대다 구조 → 일대일 소개팅보다 덜 어색
- 콘텐츠 → 대화 이끌어야 하는 부담 없음
- 자연스러운 노출 시간 → 다양한 상황에서 상대 파악
- 감정의 기복 (이기고 지고, 협력) → 사람 파악에 최적 환경

# 4. 서비스 구조

## 기본 포맷

| 항목 | 내용 |
|---|---|
| 구성 | 6~10명 (남녀 동수) |
| 빈도 | 주 1~2회 |
| 시간 | 2~3시간 |
| 활동 유형 | 보드게임 / 배드민턴 / 봉사활동 / 쿠킹클래스 등 |
| 후속 | 강제 매칭 없음 — 자율 연결 |

## 활동 카테고리

| 카테고리 | 예시 활동 | 특징 |
|---|---|---|
| 스포츠 | 배드민턴, 볼링, 풋살 | 팀 구성 자연스러움, 감정 기복 발생 |
| 게임 | 보드게임, 방탈출 | 협력/경쟁, 성격 파악 용이, 저비용 |
| 사회기여 | 봉사활동 | 가치관 공유, 진지한 이미지 |
| 문화 | 쿠킹클래스, 칵테일 만들기 | 활동 후 식사로 자연스럽게 연결 |

## 포맷 핵심 원칙

```
목적(연애)은 배경에 깔림  →  불편하지 않음
활동이 대화를 대신함    →  피로감 없음
다양한 상황 노출       →  사람 파악 깊이 있음
강제 매칭 없음         →  자연스러운 연결
```

# 5. 고객 의사결정 구조

참가자가 서비스를 선택하는 3단계

## Stage 1 — 문제 인식

"소개팅은 피곤하다, 데이팅앱은 별로다"

콘텐츠 역할

- SNS에서 "자연스러운 만남 경험" 콘텐츠 노출
- 지인 후기, 참가자 리뷰

목적: 인지 + 관심 유발

---

## Stage 2 — 서비스 탐색

"어떤 사람들이 오는지, 분위기가 어떤지 궁금하다"

콘텐츠 역할

- 실제 모임 사진/영상 (참가자 동의 기반)
- 활동 유형 설명
- 참가자 후기

목적: 신뢰 형성

---

## Stage 3 — 참가 결정

"한번 해볼까? 별로면 안 가면 되지"

콘텐츠 역할

- 쉬운 신청 프로세스
- 단건 참가 옵션 (부담 없음)
- 환불 정책 명확화

목적: 첫 참가 전환

# 8. Phase 0 — 앱 전 오프라인 실험 로드맵

> 목표: 3개월, 10회 모임, 최소 80명 참가자로 PMF 검증

## Step 1. 포맷 설계 (Week 1~2)

- 모임 규칙 확정 (인원, 남녀비, 시간, 활동 유형)
- 신청/관리 방식 결정: 구글폼 + 카카오오픈채팅 (초기)
- 첫 활동 선택: 보드게임 권장 (저비용, 장소 수급 쉬움)
- 가격 정책: 첫 2회 무료 → 이후 실비

## Step 2. 첫 모임 (Month 1 — 2회)

- 지인 네트워크로 8명 모집
- HS 직접 운영하면서 어색한 부분 관찰
- 참가 후 설문 (3가지)
  1. 다시 올 의향? (1~5점)
  2. 친구에게 추천 의향? (NPS)
  3. 가장 좋았던 순간은?

## Step 3. 확장 실험 (Month 2 — 4회)

- 지인 외 채널 모집 테스트 (인스타그램 or 커뮤니티 게시글)
- 활동 다양화: 보드게임 + 스포츠 1회
- 유료화 테스트: 실비 징수
- 자연 연결 사례 추적 (실제로 연락 이어지는지 확인)

## Step 4. 검증 및 판단 (Month 3 — 4회)

- 재참가율 측정
- 유료 참가 전환율 측정
- 자연 연결 성공률 추적
- HS 직접 운영 vs 위임 가능성 판단

---

## Go/No-go 기준

| 지표 | 목표 |
|---|---|
| 재참가율 | 50% 이상 |
| NPS | 40+ |
| 유료 전환율 | 70% 이상 |
| 자연 연결 사례 | 3건 이상 |

→ 4개 중 3개 달성 시 Phase 1 (정식 운영) 진행
→ 미달 시 포맷 피벗 or 중단

# 사업의 본질

**데이팅 앱이 아니다.**

**오프라인 소셜 경험 설계 사업이다.**

앱은 수단이고, 핵심은 **자연스러운 만남이 일어나는 환경을 만드는 것**이다.

핵심 성공 요소

1. 연애 목적 없어도 재미있는 모임
2. 연애 목적 있어도 부담 없는 구조
3. 자연스러운 후속 연결이 실제로 일어나는 것
4. 운영자 없어도 돌아가는 커뮤니티 자생 구조

이 네 가지가 동시에 작동하면 PMF다.
