# BM 설계

## 기본 원칙

위피(인앱 화폐)와 Inner Circle(심사+구독) 중간 포지션.
초기는 **참가비 단건 모델**로 시작 → 수요 검증 후 **구독 모델** 전환.

## Phase 1 — 참가비 모델 (오프라인)

| 플랜 | 가격 | 내용 | 벤치마크 |
|---|---|---|---|
| 단건 참가 | 3~5만원/회 | 1회 모임 (2~3시간, 8~10명) | 프립 활동 단가 기준 |
| 월정액 | 13만원/월 | 월 3회 참가 (4.3만/회) | 트레바리 6.25만/월 대비 저렴 |
| 프리미엄 | 25만원/월 | 월 4회 + 활동 선택권 + 참가자 구성 우선권 | Inner Circle 포지션 |

## Phase 2 — 앱 구독 모델

| 플랜 | 가격 (월) | 무료 vs 유료 경계 |
|---|---|---|
| 무료 | 0원 | 프로필 탐색, 모임 목록 조회 |
| Basic | 9,900원 | 모임 신청 1회/월, 매칭 알림 |
| Plus | 29,900원 | 모임 신청 무제한, 참가자 사전 확인 |
| Premium | 59,900원 | 1:1 매칭 큐레이션, 우선 입장 |

→ 인앱 화폐 추가 고려 (위피 젤리 벤치마크): "포인트"로 추가 신청, 프로필 열람 등 마이크로트랜잭션 가능

## 단위 경제학 (Phase 1 기준)

모임 1회 (8명 / 단건 4만원)

- 매출: 4만원 × 8명 = 32만원
- 비용: 장소 5만원 + 운영 1만원 = 6만원
- **순이익: 26만원/회**

주 2회 × 4주 = 월 8회 운영

- **월 순이익: 208만원**

호스트 1명 고용 시 (파트타임, 월 80만원)

- **월 순이익: 128만원**

# 로드맵 (가속화 버전)

> v1 대비: Phase 0 단축, 앱 출시 타이밍 앞당김

## 전체 타임라인

```
Week 1-2    포맷 + 운영 준비
Week 3-4    첫 모임 3회 (지인 기반, 무료)
Month 2     유료 전환 + 채널 오픈
Month 3     정식 브랜드 론칭
Month 4-6   호스트 채용 + 운영 확장
Month 7-9   앱 MVP 개발
Month 10    앱 런칭
```

---

## Week 1-2 — 준비

**목표**: 첫 모임 실행 가능한 최소 세팅

- [ ] 모임 규칙 확정 (인원, 성비, 활동 유형)
- [ ] 신청/관리 도구: 구글폼 + 카카오오픈채팅
- [ ] 첫 활동 선정: 보드게임 (저비용, 장소 수급 쉬움)
- [ ] 심사 기준 초안 작성 (나이, 직장, 목적)
- [ ] 인스타그램 계정 개설 (비공개 운영)

---

## Week 3-4 — 첫 모임 3회 (무료 검증)

**목표**: 포맷 검증, 어색한 부분 직접 관찰

- 지인 네트워크로 8~10명 모집
- 모임 3회 실행 (주 1~2회)
- 매 모임 후 설문 3가지
  1. 다시 올 의향? (1~5)
  2. NPS (친구 추천 의향?)
  3. 가장 좋았던 순간?
- 자연 연결 사례 추적 (연락 이어지는지)

**Go 기준**: 재참가 의향 80%+ → Month 2로 진행

---

## Month 2 — 유료 전환 + 채널 공개

**목표**: 첫 유료 매출 + 외부 모집 테스트

- 단건 3~5만원 유료 전환
- 인스타그램 첫 게시물 (실제 모임 사진, 참가자 동의 기반)
- 외부 채널 모집 테스트 (인스타 DM, 소모임 앱 공유)
- 활동 다양화: 보드게임 + 스포츠 1회 추가

**측정**

| 지표 | 목표 |
|---|---|
| 유료 참가 전환율 | 70%+ |
| 외부 모집 전환율 | 신청 대비 30%+ |
| NPS | 40+ |

---

## Month 3 — 정식 브랜드 론칭

**목표**: 브랜드 수립 + 월정액 모델 테스트

- 서비스명 확정 + 브랜딩
- 인스타그램 공개 전환 + 콘텐츠 캘린더 (주 2~3회 업로드)
- 예약 시스템 도입 (아임웹 or 노션 예약폼)
- 월정액 13만원 플랜 오픈 (베타 할인: 10만원)
- PR: 29cm, 오늘의집 커뮤니티, 에브리타임 등 타깃 채널 시딩

---

## Month 4-6 — 운영 확장

**목표**: 주 3~4회 운영 체계 구축

- 파트타임 호스트 1명 채용 (or 위탁 호스트 구조)
- 활동 다양화 (스포츠·문화·봉사 로테이션)
- 성비 밸런스 관리 시스템 (신청 필터링)
- 월 매출 목표: 500만원 이상
- KPI: 재참가율 50%+, 월정액 전환 30%+

---

## Month 7-9 — 앱 MVP 개발

**목표**: 오프라인 검증된 포맷을 앱으로 구현

**MVP 최소 기능**

- 프로필 생성 (심사 기반 입장)
- 모임 캘린더 + 신청
- 참가자 목록 사전 확인 (유료)
- 후기/평점 시스템

**기술 스택 (ARK Point 기존 역량 활용)**

- Frontend: Next.js / React
- Backend: NestJS
- DB: PostgreSQL

**개발 리소스**: 신종환(Freddie) + 외부 파트타임 1명 (or Codex 병렬 개발)

---

## Month 10 — 앱 런칭

**목표**: 오프라인 유저를 앱으로 이주

- 기존 참가자 앱 온보딩 (얼리버드 혜택)
- 앱스토어 등록 (iOS + Android)
- 구독 모델 전환 (Basic 9,900원 / Plus 29,900원)
- PR: 앱 런칭 바이럴 캠페인

# 사업의 본질

**데이팅 앱이 아니다.**

**자연스러운 만남이 일어나는 환경을 설계하는 사업이다.**

Inner Circle이 심사로 유저 퀄리티를 관리하듯,
우리는 활동으로 만남의 품질을 관리한다.

핵심 성공 요소

1. 참가자 퀄리티 관리 (심사 기반 입장)
2. 활동 설계 (어색함 해소 + 사람 파악 촉진)
3. 성비 밸런스 유지
4. 자연 연결이 실제로 일어나는 비율

이 네 가지가 동시에 작동하면 PMF다.
